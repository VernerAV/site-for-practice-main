# README – Веб-сайт ГБУ «Жилищник района Строгино» с модулем автоматизации обращений

Этот проект представляет собой адаптивный веб-сайт для Государственного бюджетного учреждения города Москвы «Жилищник района Строгино». Сайт автоматизирует приём и обработку обращений граждан, предоставляет информационные разделы (новости, платные услуги, контакты), личный кабинет жителя и панель администратора.

## Основные возможности

### Для всех посетителей (гостей)
- Просмотр новостей (слайдер на главной + отдельная страница новостей).
- Просмотр прайс-листа платных услуг (таблица с поиском).
- Контактная информация и форма подачи заявки (без регистрации).
- Адаптивный дизайн для компьютеров, планшетов и телефонов (ширина от 320 пикселей).
- Версия для слабовидящих (высокий контраст, увеличение шрифта).

### Для авторизованных жителей
- Личный кабинет с редактированием профиля (ФИО, адрес, телефон, дата рождения).
- История своих заявок с отображением статуса и ответов администратора.
- Создание новой заявки с выбором категории и предпочтительной датой.
- Уведомления об изменении статуса.

### Для администратора
- Полное управление новостями (CRUD).
- Управление услугами (прайс-лист).
- Управление пользователями (добавление, редактирование, блокировка, назначение ролей).
- Обработка заявок и сообщений (фильтрация, поиск, массовое удаление, ответы).
- Статистика по обращениям.
- Автоматическое распределение заявок по категориям/исполнителям.
- Журнал изменений.

### Мобильная версия
- Гамбургер-меню (выезжает справа при ширине ≤768px).
- Поиск под шапкой.
- Таблицы превращаются в карточки с подписями (data-label).
- Слайдер новостей на мобильных переключается в ручной режим (автопрокрутка отключена).

## Установка и запуск

### Требования
- Веб-сервер Apache с поддержкой PHP 7.4+
- MySQL 5.7+
- Расширения PHP: PDO MySQL, session, json, fileinfo

### Инструкция

1. **Клонировать репозиторий**
   ```bash
   git clone https://github.com/VernerAV/site-for-practice-main.git
   ```

2. **Создать базу данных**  
   Импортировать файл `database.sql` (если есть в репозитории; при отсутствии – создать таблицы по описанию в отчёте).

3. **Настроить подключение к БД**  
   Скопировать `includes/config.example.php` в `includes/config.php` и указать свои параметры:
   ```php
   define('DB_HOST', 'localhost');
   define('DB_NAME', 'имя_бд');
   define('DB_USER', 'пользователь');
   define('DB_PASS', 'пароль');
   ```

4. **Настройка веб-сервера**  
   Поместить проект в корень локального сервера (например, `htdocs` для XAMPP).  
   Убедиться, что модуль `mod_rewrite` включён (если используются ЧПУ).

5. **Права на папки**  
   Папка `uploads/news/` должна быть доступна для записи (chmod 755).

6. **Запуск**  
   Открыть в браузере `http://localhost/site-for-practice-main/`.

## База данных

Основные таблицы:

- `users` – учётные записи (email, пароль, роль, активен)
- `user_profiles` – профили (ФИО, дата рождения, адрес, телефон)
- `news` – новости
- `services` – платные услуги
- `message` – заявки от гостей
- `user_requests` – заявки от авторизованных пользователей
- `categories` – категории обращений
- `department_rules` – правила распределения заявок
- `assignment_log` – журнал назначений
- `notifications` – уведомления

Полная ER-диаграмма и SQL-скрипты приведены в отчёте по практике (раздел 2.5).
